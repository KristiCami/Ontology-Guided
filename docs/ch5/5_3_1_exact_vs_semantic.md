# 5.3.1 Exact vs Semantic

Η αξιολόγηση του OG–NSD στηρίζεται στη σύγκριση παραγόμενων axioms με το gold ontology. Κρίσιμο σημείο είναι η διάκριση ανάμεσα σε exact (συντακτική) και semantic (εννοιολογική) αντιστοίχιση. Η exact αξιολόγηση θεωρεί επιτυχία μόνο όταν το παραγόμενο axiom είναι ταυτόσημο με το gold σε επίπεδο IRI. Για παράδειγμα, ένα `:Withdrawal rdfs:subClassOf :Transaction` θεωρείται σωστό μόνο αν το gold έχει ακριβώς το ίδιο triple. Αυτή η προσέγγιση είναι αυστηρή, αλλά παρέχει καθαρή μέτρηση precision/recall ως προς τη συγκεκριμένη οντολογία αναφοράς.

Η semantic αξιολόγηση, αντίθετα, βασίζεται στην έννοια του entailment. Ένα axiom μπορεί να θεωρηθεί σωστό αν προκύπτει λογικά από την παραγόμενη οντολογία, ακόμη και αν δεν είναι ρητά καταγεγραμμένο. Για παράδειγμα, ένα axiom που έπεται από transitive subclass chains ή από domain/range constraints μπορεί να αξιολογηθεί ως “σωστό” σε semantic επίπεδο. Το README περιγράφει ότι η αξιολόγηση λαμβάνει υπόψη τέτοιες semantic αντιστοιχίσεις, ιδίως όταν ενεργοποιείται reasoning. Αυτό επιτρέπει πιο δίκαιη αποτίμηση της ποιότητας του LLM, αφού αποτυπώνει όχι μόνο την ικανότητα ακριβούς αναπαραγωγής αλλά και την ικανότητα να παράγει λογικά συνεπή δομές.

Στην πράξη, οι δύο προσεγγίσεις οδηγούν σε διαφορετικά metrics. Η exact αξιολόγηση δίνει συνήθως υψηλότερη precision (λιγότερα false positives) αλλά χαμηλότερη recall, επειδή δεν “πιστώνει” τις έμμεσες συνέπειες. Η semantic αξιολόγηση αυξάνει την recall, καθώς αναγνωρίζει περισσότερα σωστά axioms, αλλά μπορεί να μειώσει precision αν το model δημιουργεί λογικά ισοδύναμες αλλά μη επιθυμητές δομές. Για το OG–NSD, η παρουσία SHACL shapes και reasoning βοηθά να μειωθούν οι semantic αποκλίσεις, έτσι ώστε η semantic αξιολόγηση να μην διογκώνει λανθασμένα αποτελέσματα.

Σε ακαδημαϊκό πλαίσιο, είναι σημαντικό να αναφερθεί ρητά ποια μορφή αξιολόγησης χρησιμοποιείται σε κάθε πίνακα ή γράφημα. Το pipeline χρησιμοποιεί macro- και micro-F1 ανά axiom type (Classes, SubClassOf, Domain, Range, ObjectProperty, DatatypeProperty) και μπορεί να υπολογιστεί τόσο σε exact όσο και σε semantic επίπεδο. Η σύγκριση αυτών των δύο βοηθά να δείξει αν το σύστημα βελτιώνεται όχι μόνο ως προς την “αντιγραφή” του gold, αλλά και ως προς τη λογική ορθότητα του τελικού γράφου.

Συνολικά, η διάκριση exact vs semantic είναι κρίσιμη για την ορθή ερμηνεία των αποτελεσμάτων. Η exact αξιολόγηση είναι πιο αυστηρή και κατάλληλη για benchmarking, ενώ η semantic είναι πιο κοντά στην πραγματική χρήση, όπου αυτό που έχει σημασία είναι η ικανότητα του ontology να υποστηρίζει λογικά σωστά συμπεράσματα. Για την τεκμηρίωση της υλοποίησης, πρέπει να γίνει σαφές ότι το OG–NSD υποστηρίζει και τις δύο οπτικές, και ότι οι επιλογές αξιολόγησης διαμορφώνουν το πώς ερμηνεύονται τα πειραματικά αποτελέσματα.
