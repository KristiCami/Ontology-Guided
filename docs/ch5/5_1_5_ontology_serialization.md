# 5.1.5 Ontology Serialization

Η τελική έξοδος του OG–NSD είναι ένα πλήρως σειριοποιημένο OWL ontology σε Turtle (TTL), κατάλληλο για επαναχρησιμοποίηση ή αξιολόγηση με εξωτερικά εργαλεία. Το serialization υλοποιείται στο `og_nsd/ontology.py`, το οποίο λειτουργεί ως layer πάνω από `rdflib` για τη δημιουργία, συγχώνευση και αποθήκευση γράφων RDF. Ο pipeline συνθέτει το τελικό γράφημα από τρεις πηγές: (α) το draft που παρήγαγε ο LLM, (β) τυχόν base ontology που δίνεται μέσω `--base`, και (γ) τα repair patches που προστίθενται σε κάθε iteration. Η συγχώνευση αυτών των πηγών γίνεται με τρόπο που διατηρεί τη μοναδικότητα των triples και εξασφαλίζει ότι όλα τα namespaces παραμένουν συνεπή.

Στο serialization περιλαμβάνονται επίσης namespaces και prefixes. Το pipeline χρησιμοποιεί τις πληροφορίες prefix τόσο από το base όσο και από το LLM output, ώστε το τελικό TTL να έχει τυπική δομή και να είναι άμεσα αναγνώσιμο από reasoners και SHACL engines. Η επιλογή Turtle είναι σημαντική επειδή είναι ανθρώπινα αναγνώσιμη, διευκολύνει την ποιοτική ανάλυση σε ακαδημαϊκό πλαίσιο, και επιτρέπει άμεση αξιολόγηση με εργαλεία όπως `pyshacl` και `owlready2`.

Κατά τη διάρκεια του repair loop, κάθε iteration μπορεί να αποθηκεύει προσωρινές εκδόσεις του γράφου, κάτι που επιτρέπει την ανάλυση της εξέλιξης. Τα τελικά outputs εμφανίζονται στο `build/` για ad-hoc runs ή στο `runs/` για επίσημα experiments. Έτσι, ένας ερευνητής μπορεί να παρακολουθήσει τη διαφορά ανάμεσα στο αρχικό draft, στις ενδιάμεσες διορθώσεις και στο τελικό ontology. Αυτό είναι κρίσιμο για επιστημονική τεκμηρίωση, καθώς τεκμηριώνει τη διαδρομή από το αρχικό input έως την τελική μορφή του ontology.

Το serialization υποστηρίζει επίσης την παραγωγή αναφορών. Το `og_nsd/reporting.py` δημιουργεί JSON report που συνοδεύει το TTL. Το report περιλαμβάνει συνοπτικά metrics (π.χ. αριθμό violations ανά iteration), CQ αποτελέσματα, και logging των LLM prompts/outputs. Έτσι, το τελικό artefact δεν είναι μόνο το TTL αλλά ένα ζεύγος (TTL + report) που επιτρέπει πλήρη ανακατασκευή της διαδικασίας.

Στο πλαίσιο ακαδημαϊκής συγγραφής, η ενότητα “Ontology Serialization” εξηγεί ότι το σύστημα δεν παράγει απλώς κείμενο, αλλά δημιουργεί ένα τυποποιημένο, συμβατό με W3C standards γράφημα. Η ύπαρξη του base merge και των patch commits αποδεικνύει ότι το σύστημα υποστηρίζει incremental editing και δεν αντικαθιστά αυθαίρετα το ontology. Επομένως, το τελικό TTL αντιπροσωπεύει το αποτέλεσμα ενός ελεγχόμενου, επαναλήψιμου pipeline.
