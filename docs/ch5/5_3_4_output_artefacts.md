# 5.3.4 Output Artefacts

Η αξιολόγηση του OG–NSD δεν καταλήγει μόνο σε αριθμητικά metrics αλλά και σε ένα σύνολο artefacts που τεκμηριώνουν πλήρως την εκτέλεση. Τα βασικά artefacts είναι: (α) το τελικό Turtle ontology (`.ttl`), (β) τα JSON reports ανά run, και (γ) ενδιάμεσες εκδόσεις του ontology ανά iteration (όταν ενεργοποιείται το repair loop). Τα αρχεία αυτά αποθηκεύονται είτε στον φάκελο `build/` για ad-hoc runs είτε στο `runs/` για τα επίσημα experiments. Η δομή αυτή επιτρέπει την πλήρη αναπαραγωγή της διαδικασίας, καθώς κάθε output είναι συσχετισμένο με συγκεκριμένο config και script.

Το τελικό Turtle αρχείο είναι η πιο άμεση μορφή αποτελέσματος: χρησιμοποιείται σε downstream εργαλεία, όπως reasoners ή visualization frameworks. Το JSON report, που δημιουργείται από το `og_nsd/reporting.py`, περιλαμβάνει λεπτομερείς πληροφορίες για κάθε iteration: αριθμό SHACL violations, CQ pass rates, πιθανές unsat classes, και σύνοψη των LLM outputs. Αυτό μετατρέπει το report σε “logbook” της εκτέλεσης και επιτρέπει λεπτομερή ανάλυση της πορείας του pipeline.

Για τα πειράματα E1–E6, τα artefacts στο `runs/` λειτουργούν ως dataset αποτελεσμάτων. Ο ερευνητής μπορεί να συγκρίνει τα TTLs διαφορετικών experiments, να εξετάσει αν οι παραβιάσεις μειώθηκαν μεταξύ E3 (no-repair) και E4 (full), ή να δει αν οι CQs βελτιώθηκαν στο E6. Επίσης, ο φάκελος `runs/` μπορεί να χρησιμοποιηθεί για συλλογή στατιστικών σε μεταγενέστερο στάδιο, αφού τα JSON reports έχουν κοινή μορφή.

Η ύπαρξη ενδιάμεσων artefacts ανά iteration είναι κρίσιμη για την τεκμηρίωση του repair loop. Δίνει τη δυνατότητα να παρουσιαστεί ποιοτικά η βελτίωση του ontology μετά από κάθε γύρο. Για παράδειγμα, μπορεί να συγκριθεί το draft-only ontology (iteration 0) με το final ontology (iteration N) και να παρουσιαστεί η διαφορά σε SHACL violations ή CQ coverage. Αυτό προσφέρει συγκεκριμένη απεικόνιση της επίδρασης του feedback μηχανισμού.

Στην ακαδημαϊκή εργασία, τα output artefacts αποτελούν πρωτογενές υλικό τεκμηρίωσης. Επιτρέπουν όχι μόνο την αναπαραγωγή των πειραμάτων αλλά και τη διαφανή αποτίμηση των αποτελεσμάτων. Η ύπαρξη σταθερής δομής artefacts (TTL + JSON report) καθιστά την αξιολόγηση επαναληπτική και επιβεβαιώσιμη, σύμφωνα με τις απαιτήσεις επιστημονικής μεθοδολογίας.
