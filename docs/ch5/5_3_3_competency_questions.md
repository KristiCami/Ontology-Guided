# 5.3.3 Competency Questions

Οι Competency Questions (CQs) αποτελούν ένα τρίτο επίπεδο αξιολόγησης που συμπληρώνει τα SHACL και reasoning metrics. Στο OG–NSD, οι CQs υλοποιούνται ως SPARQL ASK queries, αποθηκευμένες σε `.rq` αρχεία όπως `atm_cqs.rq` και `health_cqs.rq`. Η εκτέλεσή τους γίνεται από το `og_nsd/queries.py`, το οποίο φορτώνει το CQ αρχείο, χωρίζει τα queries με βάση κενές γραμμές/σχόλια και εκτελεί κάθε ASK query πάνω στο παραγόμενο ontology. Το αποτέλεσμα είναι Boolean (true/false) και καταγράφεται στο report ανά iteration.

Οι CQs λειτουργούν ως “domain adequacy” test: δεν ελέγχουν μόνο τη δομή αλλά και το αν το ontology απαντά σε βασικά ερωτήματα της εφαρμογής. Για παράδειγμα, σε ATM domain μια CQ μπορεί να ελέγχει αν κάθε Withdrawal έχει ποσό ή αν μια Transaction συνδέεται με λογαριασμό. Αν αυτά τα queries αποτυγχάνουν, σημαίνει ότι η οντολογία δεν καλύπτει κρίσιμες απαιτήσεις, ακόμη κι αν είναι SHACL-valid. Έτσι, οι CQs αξιολογούν την εννοιολογική πληρότητα και την χρηστικότητα του ontology.

Στο pipeline, τα CQ αποτελέσματα μπορούν να επηρεάσουν την επιλογή repairs, ειδικά στο E6 “CQ-oriented” experiment. Η ιδέα είναι ότι το repair loop δεν διορθώνει μόνο SHACL violations αλλά μπορεί να στοχεύει και σε αποτυχίες CQs. Σε αυτή την εκδοχή, κάθε iteration καταγράφει την CQ pass rate, επιτρέποντας ανάλυση του τρόπου με τον οποίο η πληρότητα των απαντήσεων βελτιώνεται σταδιακά. Έτσι, οι CQs λειτουργούν ως αξιολόγηση προστιθέμενης αξίας του repair loop πάνω από την απλή συμμόρφωση.

Η αναφορά των CQs στην ακαδημαϊκή τεκμηρίωση πρέπει να περιλαμβάνει τόσο τη διαδικασία δημιουργίας τους όσο και τον ρόλο τους στα πειράματα. Οι CQs είναι χειροκίνητα σχεδιασμένες με βάση τις απαιτήσεις και λειτουργούν ως “τεστ λειτουργικότητας” της οντολογίας. Στο OG–NSD, οι CQ αναφορές ενσωματώνονται στο JSON report, ώστε να υπάρχει διαφανής καταγραφή για κάθε run. Αυτό επιτρέπει σύγκριση μεταξύ E1–E6, αλλά και ανάμεσα σε διαφορετικά domains.

Συνολικά, οι CQs δεν αντικαθιστούν την SHACL ή τον reasoning έλεγχο, αλλά τους συμπληρώνουν. Ενώ τα SHACL/Reasoner metrics αφορούν δομική/λογική ορθότητα, οι CQs αφορούν “domain coverage” και επιχειρησιακή επάρκεια. Αυτό το τρίτο επίπεδο αξιολόγησης είναι κρίσιμο για ακαδημαϊκή εργασία, γιατί δείχνει ότι το σύστημα δεν παράγει απλώς τυπικά σωστά ontologies αλλά και λειτουργικά χρήσιμα.
