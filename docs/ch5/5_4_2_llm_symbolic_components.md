# 5.4.2 LLM + Symbolic Components

Το OG–NSD είναι ένα καθαρό παράδειγμα νευρο–συμβολικού συστήματος, όπου το LLM λειτουργεί ως γεννήτρια προτάσεων και τα συμβολικά modules επιβάλλουν έλεγχο και διόρθωση. Η συνεργασία αυτή δομείται γύρω από τέσσερα βασικά components: (α) LLM drafting (`og_nsd/llm.py`), (β) ontology assembly/serialization (`og_nsd/ontology.py`), (γ) SHACL validation (`og_nsd/shacl.py`), και (δ) DL reasoning (`og_nsd/reasoning.py`). Η pipeline κεντρική λογική (`og_nsd/pipeline.py`) συντονίζει αυτά τα components σε έναν βρόχο όπου η νευρωνική παραγωγή και η συμβολική επαλήθευση αλληλεπιδρούν.

Το LLM component παράγει Turtle axioms από φυσική γλώσσα. Η παραγωγή μπορεί να είναι είτε πραγματικό LLM output (OpenAI backend) είτε heuristic fallback για αναπαραγωγιμότητα. Το κείμενο που επιστρέφει το LLM δεν θεωρείται “έγκυρο” μέχρι να περάσει από το symbolic layer. Το `og_nsd/ontology.py` μετατρέπει το κείμενο σε RDF graph και το συγχωνεύει με υπάρχον base ontology.

Το SHACL module χρησιμοποιεί τα shape constraints ως κανόνες εγκυρότητας. Η επιστροφή από το `pyshacl` περιλαμβάνει αναλυτικά violations, που καταγράφονται και μετατρέπονται σε prompts. Αυτό το βήμα είναι η “γέφυρα” μεταξύ συμβολικής επαλήθευσης και νευρωνικής διόρθωσης. Ο reasoner, όταν ενεργοποιηθεί, λειτουργεί ως δεύτερο επίπεδο, εξασφαλίζοντας ότι το ontology είναι συνεπές σε επίπεδο OWL. Η ενσωμάτωση reasoning είναι προαιρετική για να μην αυξάνεται υπέρμετρα το runtime, αλλά προσφέρει ισχυρότερη εγγύηση ορθότητας.

Το pipeline υλοποιεί την αλληλεπίδραση ως εξής: ο LLM παράγει draft, το symbolic layer αξιολογεί και επιστρέφει violations, ο LLM λαμβάνει αυτά τα violations ως structured prompt και προτείνει patches. Οι patches ενσωματώνονται και η διαδικασία επαναλαμβάνεται μέχρι conformance ή εξάντληση iterations. Αυτή η διαδικασία είναι θεμελιώδης για το E4 experiment και θεωρείται η κύρια συνεισφορά του συστήματος.

Η ακαδημαϊκή τεκμηρίωση πρέπει να τονίζει ότι το σύστημα δεν βασίζεται μόνο σε LLM παραγωγή ούτε μόνο σε symbolic heuristics. Ο συνδυασμός επιτρέπει υψηλότερη κάλυψη (λόγω LLM) και υψηλότερη ακρίβεια/συμμόρφωση (λόγω SHACL/Reasoner). Η υβριδική αυτή προσέγγιση είναι καινοτόμος στο πλαίσιο ontology drafting, επειδή μετατρέπει τις symbolic παραβιάσεις σε ενεργή καθοδήγηση του LLM, δημιουργώντας μια πρακτική μορφή “feedback learning” χωρίς εκπαίδευση μοντέλου.

Συνολικά, η ενότητα αυτή πρέπει να παρουσιάσει το OG–NSD ως σύστημα όπου οι δύο συνιστώσες δεν λειτουργούν ανεξάρτητα, αλλά ως συνδεδεμένα υποσυστήματα με σαφή ρόλο και ανταλλαγή πληροφορίας. Η τεκμηρίωση αυτής της συνεργασίας είναι κρίσιμη για να εξηγηθεί γιατί το σύστημα υπερτερεί έναντι των LLM-only ή symbolic-only baselines.
